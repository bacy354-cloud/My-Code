# Happy-LLM 第三章：预训练语言模型笔记

## 本章定位

第三章的核心不是背每个模型的全部细节，而是理解一句话：

```text
预训练语言模型 = Transformer 不同部件 + 不同预训练任务 + 不同下游使用方式。
```

本章最重要的三条主线是：

```text
Encoder-only      -> BERT -> 偏理解
Encoder-Decoder   -> T5   -> 输入文本转输出文本
Decoder-only      -> GPT  -> 偏生成
```

第三章暂时不需要把 RoBERTa、ALBERT、GLM、LLaMA 的所有细节都细抠。它们更适合作为主线模型的补充：

```text
RoBERTa / ALBERT：BERT 系改进
LLaMA：GPT-style Decoder-only 的现代工程增强版
GLM：试图融合 MLM 和 CLM 的空白填充路线
```

## 0. 学习检测结果

本章小测已通过。

已经掌握的点：

- BERT 使用 Encoder-only，偏理解任务。
- T5 使用 Encoder-Decoder，把任务统一成 text-to-text。
- GPT 使用 Decoder-only，偏自回归生成。
- BERT 的 MLM 是随机挖空，利用左右上下文预测被遮住的 token。
- GPT 的 CLM 是只看前文，预测下一个 token。
- T5 的 cross-attention 中，Q 来自 Decoder，K/V 来自 Encoder output。
- GPT 没有 Encoder，所以没有 cross-attention，Q/K/V 都来自当前 hidden states。
- LLaMA 和 GPT 是同一条 Decoder-only 主线，主要是工程细节现代化。
- GLM 的核心是自回归空白填充，试图融合 MLM 和 CLM。

需要修正和强化的点：

```text
1. BERT 不是“完全没有 mask”，而是没有 causal mask；它仍然可能有 padding mask，MLM 里还有 [MASK] token。
2. BERT 不适合自由续写，不是因为“没有生成层”这么简单，而是因为它是双向 Encoder，不按自回归方式逐 token 解码。
3. MLM 是随机遮输入中的 token 或片段，不一定只遮句中。
4. CLM 不是把未来 token 替换成 [MASK]，而是用 causal mask 禁止当前位置看到未来。
5. GPT 训练时 logits[:, i, :] 预测 input_ids[:, i+1]；推理时 logits[:, -1, :] 用来预测当前上下文的下一个 token。
6. SiLU(x) = x * sigmoid(x)，不是 x + sigmoid(x)。
7. GQA / repeat_kv 的关键收益不只是减少参数，更重要是减少推理时 KV Cache 的显存和带宽压力。
```

## 1. 三类预训练语言模型

Transformer 原版由 Encoder 和 Decoder 组成：

```text
Transformer = Encoder + Decoder
```

后来的预训练语言模型根据使用部件不同，分成三条路线：

| 路线 | 代表模型 | 使用的 Transformer 部件 | 主要能力 | 典型训练目标 |
|---|---|---|---|---|
| Encoder-only | BERT / RoBERTa / ALBERT | Encoder | 理解、分类、匹配、抽取 | MLM / NSP / SOP |
| Encoder-Decoder | T5 | Encoder + Decoder | 输入文本转输出文本 | 去噪生成 / text-to-text |
| Decoder-only | GPT / LLaMA / GLM | Decoder 主体 | 续写、生成、对话 | CLM / 空白填充 |

一句话区分：

```text
BERT 像阅读理解器。
T5 像文本转换器。
GPT / LLaMA 像通用续写器。
```

## 2. Encoder-only：BERT 系

BERT 解决的问题是：

```text
给我一段文本，我要理解它，然后做判断、匹配、抽取或分类。
```

所以 BERT 只使用 Encoder：

```text
input_ids
-> Embedding
-> 多层 Encoder
-> hidden_states
-> prediction head
```

形状：

```text
input_ids:     [B, T]
hidden_states: [B, T, C]
```

如果是分类任务，常用 `[CLS]` 位置作为整句表示：

```python
cls_hidden = hidden_states[:, 0, :]  # [B, C]
logits = classifier(cls_hidden)      # [B, num_labels]
```

BERT 的关键是双向理解：

```text
每个 token 可以同时看左边和右边。
```

因此 BERT 不使用 causal mask。它不是自回归生成模型。

### 2.1 MLM

MLM 是 Masked Language Model，解决的问题是：

```text
如何让模型利用左右上下文，预测被挖掉的 token？
```

例子：

```text
输入：我 喜欢 [MASK] 学习
目标：机器
```

BERT 可以同时看：

```text
左边：我 喜欢
右边：学习
```

所以它学到的是双向语义。

形状：

```text
input_ids: [B, T]
hidden:    [B, T, C]
logits:    [B, T, V]
```

但 MLM 的 loss 通常只在被选中的预测位置上计算。

BERT 原始策略大致是随机选 15% token 做预测目标：

```text
80% 替换成 [MASK]
10% 替换成随机 token
10% 保持原 token 不变
```

这样做是为了减轻预训练和真实下游任务之间的差异，因为真实下游文本里通常不会到处出现 `[MASK]`。

### 2.2 NSP

NSP 是 Next Sentence Prediction，解决的问题是：

```text
模型能不能判断两个句子是否相邻或相关？
```

例子：

```text
Sentence A: 我今天学习了 BERT。
Sentence B: 它是一种 Encoder-only 模型。
标签：IsNext
```

NSP 更偏句子关系能力，常用 `[CLS]` 向量接二分类头：

```text
[CLS] hidden -> classifier -> IsNext / NotNext
```

### 2.3 RoBERTa

RoBERTa 的核心不是换架构，而是把 BERT 训练得更充分：

```text
RoBERTa = BERT 架构
+ 去掉 NSP
+ 动态 Mask
+ 更多数据、更大 batch、更久训练
```

动态 Mask 的意思是，同一个样本在不同训练轮次可以遮不同位置：

```text
第 1 次：我 喜欢 [MASK] 学习
第 2 次：我 [MASK] 机器 学习
第 3 次：[MASK] 喜欢 机器 学习
```

这样同一个样本能提供更多训练信号。

### 2.4 ALBERT

ALBERT 解决的问题是：

```text
BERT 参数太大，能不能减少参数但尽量保留能力？
```

主要思路：

```text
1. Embedding 参数分解
2. 多层 Encoder 参数共享
3. 用 SOP 替代 NSP
```

参数共享减少的是参数量，不一定大幅减少推理计算量。因为模型仍然要按层执行计算，只是多层共用同一套参数。

## 3. Encoder-Decoder：T5

T5 解决的问题是：

```text
能不能把所有 NLP 任务都统一成一种格式？
```

它的答案是：

```text
Text-to-Text
输入文本 -> 输出文本
```

所以 T5 不管做分类、翻译、摘要还是问答，都统一成生成文本。

例子：

```text
情感分类
输入：sentiment: 这部电影很好看
输出：positive
```

```text
翻译
输入：translate English to Chinese: I love NLP
输出：我喜欢自然语言处理
```

```text
摘要
输入：summarize: 一大段文章
输出：三句话摘要
```

### 3.1 T5 的结构

T5 使用完整的 Encoder-Decoder：

```text
Encoder：读输入
Decoder：生成输出
cross-attention：Decoder 查询 Encoder 的语义信息
```

形状：

```text
encoder_input_ids: [B, S]
decoder_input_ids: [B, T]

encoder_hidden: [B, S, C]
decoder_hidden: [B, T, C]

cross_attn_score: [B, H, T, S]
logits:           [B, T, V]
```

cross-attention 中：

```text
Q 来自 Decoder 当前状态
K 来自 Encoder output
V 来自 Encoder output
```

所以注意力分数不是 `[T, T]`，而是：

```text
[T, S]
```

含义是：

```text
输出端每个位置，都去看输入端所有位置。
```

### 3.2 T5 的去噪生成

T5 的预训练目标可以理解成：

```text
把一段被破坏的文本还原出来。
```

例子：

```text
原句：我 喜欢 机器 学习，因为 它 很 有趣。
输入：我 喜欢 <extra_id_0>，因为 <extra_id_1>。
输出：<extra_id_0> 机器 学习 <extra_id_1> 它 很 有趣
```

它和 BERT 的区别是：

```text
BERT：在原位置预测 [MASK]。
T5：把被挖掉的内容组织成目标序列，由 Decoder 生成出来。
```

一句话：

```text
BERT 是原地做完形填空；T5 是读被破坏的输入，再生成缺失内容。
```

## 4. Decoder-only：GPT

GPT 解决的问题是：

```text
给定前面的 token，预测下一个 token。
```

例子：

```text
输入：我 今天 学习
目标：GPT
```

GPT 是 Decoder-only，但它不是完整原始 Transformer Decoder。原始 Decoder 有：

```text
masked self-attention
cross-attention
FFN
```

GPT 没有 Encoder，所以没有 cross-attention，只保留：

```text
masked self-attention + MLP
```

一层 GPT DecoderLayer 可以理解为：

```text
hidden_states
-> LayerNorm
-> Masked Self-Attention
-> 残差连接
-> LayerNorm
-> MLP
-> 残差连接
```

形状：

```text
input_ids: [B, T]
hidden:    [B, T, C]
Q/K/V:     [B, H, T, D]
score:     [B, H, T, T]
logits:    [B, T, V]
```

GPT 的 Q/K/V 都来自同一份 hidden states：

```text
Q = hidden_states
K = hidden_states
V = hidden_states
```

这是 self-attention，不是 T5 的 cross-attention。

### 4.1 causal mask

GPT 必须使用 causal mask，因为训练时不能偷看未来 token。

如果输入是：

```text
我 今天 学习 GPT
```

在预测“学习”时，模型只能看：

```text
我 今天
```

不能看后面的：

```text
GPT
```

mask 通常把未来位置加上 `-inf`：

```text
softmax(-inf) = 0
```

所以未来 token 的注意力权重变成 0。

### 4.2 CLM 与 shift loss

CLM 是 Causal Language Modeling：

```text
用前面的 token 预测下一个 token。
```

假设：

```text
input_ids = [我, 今天, 学, GPT]
位置        0    1    2   3
```

模型输出：

```text
logits = [
  位置 0 的预测,
  位置 1 的预测,
  位置 2 的预测,
  位置 3 的预测
]
```

对应关系是：

| logits 位置 | 已经看到 | 应该预测 |
|---|---|---|
| logits[0] | 我 | 今天 |
| logits[1] | 我 今天 | 学 |
| logits[2] | 我 今天 学 | GPT |
| logits[3] | 我 今天 学 GPT | 当前样本里没有标准答案 |

所以训练时要错位对齐：

```python
shift_logits = logits[:, :-1, :]   # 去掉最后一个预测
shift_labels = input_ids[:, 1:]    # 去掉第一个 token，当作答案
```

含义是：

```text
logits[:, 0, :] -> input_ids[:, 1]
logits[:, 1, :] -> input_ids[:, 2]
logits[:, 2, :] -> input_ids[:, 3]
```

一句话：

```text
第 i 个位置的 logits，用来预测第 i+1 个 token。
```

### 4.3 推理为什么只取最后一个 logits

推理时，当前只需要预测下一个 token。

如果上下文是：

```text
我 今天 学习 GPT
```

模型虽然会输出：

```text
logits: [B, T, V]
```

但真正需要的是最后一个位置：

```python
next_logits = logits[:, -1, :]  # [B, V]
```

因为最后一个位置代表：

```text
根据当前完整上下文，预测下一个 token。
```

训练和推理的区别：

```text
训练：已有完整序列，可以并行算多个位置的 next-token loss。
推理：只需要根据当前上下文生成下一个 token。
```

### 4.4 GPT 为什么成为 LLM 主线

GPT 后来成为 LLM 主线，核心原因不是结构最复杂，而是三件事高度统一：

```text
数据形式：任何连续文本都能作为训练数据。
训练目标：给前文，预测后文。
推理形式：给 prompt，继续生成。
```

聊天也是同一个形式：

```text
给对话历史，继续生成助手回复。
```

所以 GPT 的预训练、指令微调、聊天推理都可以保持“续写”主线。

## 5. LLaMA：现代 GPT-style Decoder-only

LLaMA 不是一条脱离 Transformer 的新路线。

更准确地说：

```text
Transformer：基础架构
GPT：Decoder-only Transformer
LLaMA：现代化 GPT-style Decoder-only Transformer
LLM：规模、数据、训练和对齐放大后的能力阶段
```

LLaMA 主线仍然是：

```text
Decoder-only + causal mask + CLM
```

它相对早期 GPT 的主要变化是工程细节：

```text
1. RMSNorm / LlamaRMSNorm
2. RoPE / rotary positional embedding
3. GQA / repeat_kv
4. SwiGLU 风格门控 MLP
```

### 5.1 RMSNorm

RMSNorm 是 LayerNorm 的变体，用来稳定训练。

直觉作用：

```text
别让 hidden_states 的数值乱飞，让深层网络更稳。
```

### 5.2 RoPE

RoPE 是旋转位置编码。

它主要作用在：

```text
Q 和 K
```

不直接作用在 V 上。

直觉作用：

```text
把位置信息注入注意力匹配过程，让模型知道 token 之间的位置关系。
```

### 5.3 GQA / repeat_kv

普通多头注意力可能是：

```text
Q heads = 32
K heads = 32
V heads = 32
```

GQA 可以是：

```text
Q heads = 32
K/V heads = 8
```

`repeat_kv` 的作用是：

```text
把较少的 K/V 头复制到多个 Q head 上，让形状匹配。
```

它的关键收益：

```text
减少推理时 KV Cache 的显存和带宽压力。
```

尤其是长上下文生成时，这个收益很明显。

### 5.4 门控 MLP

LLaMA 的 MLP 常见形式：

```python
gate = silu(gate_proj(x))
up = up_proj(x)
hidden = gate * up
out = down_proj(hidden)
```

其中：

```text
SiLU(x) = x * sigmoid(x)
```

直觉理解：

```text
up_proj：生成候选信息。
gate_proj + SiLU：决定哪些维度该通过。
gate * up：按维度筛选和调制信息。
down_proj：压回 hidden size。
```

一句话：

```text
gate_proj 像一个可学习的信息开关。
```

## 6. GLM

GLM 的核心不是换了完全不同的模型结构，而是提出一种预训练任务：

```text
Autoregressive Blank Infilling
自回归空白填充
```

它想融合：

```text
MLM 的双向理解
CLM 的自回归生成
```

例子：

```text
原句：我 今天 学习 GPT，因为 它 很 强。
挖空：我 今天 [MASK]，因为 它 很 强。
答案：学习 GPT
```

这里有两层含义：

```text
空白外部：可以看左边和右边上下文，像 BERT / MLM。
空白内部：要从左到右生成缺失片段，像 GPT / CLM。
```

所以可以记成：

```text
GLM = 在 Decoder-only 主体上，用自回归空白填充融合 MLM 和 CLM。
```

但大模型时代，纯 CLM 又逐渐成为主线。原因是：

```text
CLM 目标更简单。
CLM 数据构造更自然。
CLM 训练和推理形式更一致。
模型规模变大后，生成模型也能学到很强理解能力。
```

## 7. BERT / T5 / GPT 总对比

| 问题 | BERT | T5 | GPT / LLaMA |
|---|---|---|---|
| 架构 | Encoder-only | Encoder-Decoder | Decoder-only |
| 主要能力 | 理解 | 文本转换 | 生成 |
| 能否双向看输入 | 可以 | Encoder 可以 | 不能看未来 |
| 是否有 cross-attention | 没有 | 有 | 没有 |
| 典型目标 | MLM / NSP | 去噪生成 | CLM |
| 输出形式 | 分类、标注、抽取等 | 输出文本 | 续写文本 |
| 典型任务 | 分类、匹配、抽取问答 | 翻译、摘要、问答 | 对话、续写、代码生成 |

最核心判断：

```text
BERT：读懂后做判断。
T5：读懂输入后生成目标文本。
GPT：根据已有上下文继续生成。
```
