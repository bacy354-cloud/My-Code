# Happy-LLM 第一章：NLP 基础概念笔记

## 本章定位

第一章不是代码章节，也不是需要背诵的理论清单。它的作用是给后面的 Transformer、预训练语言模型和 LLM 铺垫一个问题：

```text
人类语言如何变成模型可以计算、可以学习、可以生成的形式？
```

本章最重要的主线是：

```text
NLP 任务
-> 文本预处理
-> 文本表示
-> 语言模型
-> 上下文表示
-> Transformer / LLM
```

## 1. NLP 解决什么问题

NLP，也就是自然语言处理，目标是让计算机能够理解、处理和生成自然语言。

可以粗略分成两类能力：

```text
理解：分类、实体识别、关系抽取、信息检索
生成：摘要、翻译、问答、对话生成
```

现在的 LLM 把很多传统 NLP 任务统一成了类似的形式：

```text
输入文本 -> 模型理解上下文 -> 输出文本
```

所以第一章不是为了学一堆零散任务，而是为了看清：LLM 是从传统 NLP 任务和语言模型一路发展过来的。

## 2. 中文分词、子词切分和 Tokenizer

中文分词更像是在“词”的层面切开句子。

```text
我喜欢大模型
-> 我 / 喜欢 / 大模型
```

子词切分会把词进一步切成更小的单位，解决生僻词、未登录词、复合词的问题。

```text
unhappy
-> un / happy
```

现代 LLM 更接近子词切分。模型通常不是直接处理完整单词，而是先通过 tokenizer 把文本切成 token，再把 token 变成数字 id。

基本链路是：

```text
文本 -> tokenizer -> token -> token id -> embedding 向量 -> 模型计算
```

## 3. 文本为什么必须表示成数字

神经网络不能直接处理汉字、英文单词或句子，它只能对数字张量做计算。

所以 NLP 的核心前置问题是：

```text
如何把文本变成数字，同时尽量保留语义、词序和上下文信息？
```

这就是“文本表示”的意义。

## 4. 文本表示的发展

### 4.1 VSM / One-hot

VSM 或 One-hot 的思想是：给词表中的每个词一个位置。

例如词表是：

```text
[我, 喜欢, 苹果, 手机]
```

句子：

```text
我 喜欢 苹果
```

可以表示为：

```text
[1, 1, 1, 0]
```

它的优点是简单，缺点也很明显：

```text
维度很高
大部分位置是 0，十分稀疏
不懂词序
不懂语义相似
```

### 4.2 N-gram

N-gram 的核心是看连续出现的 N 个词或字。

例如：

```text
我 喜欢 大模型
```

2-gram 是：

```text
我 喜欢
喜欢 大模型
```

N-gram 可以有三种常见用途：

```text
文本表示：把连续词片段当成特征
统计语言模型：用前 N-1 个词预测下一个词
评价指标：BLEU、ROUGE 等会看 N-gram 重合度
```

它和 GPT 的 next token prediction 有相似点：都在根据前文预测后文。

区别是：

```text
N-gram：只看固定长度的局部窗口，主要靠统计频率
GPT：在上下文窗口内看更长前文，用神经网络估计下一个 token 的概率
```

所以 N-gram 不是现代 LLM 的核心工具，但它是语言模型思想的早期形态之一。

### 4.3 Word2Vec

Word2Vec 把词变成低维、稠密、有语义关系的向量。

核心思想是：

```text
经常出现在相似上下文中的词，语义也往往相似。
```

例如：

```text
国王、王后、皇帝
苹果、水果、香蕉
```

这些词在向量空间中的位置会更接近。

相比 One-hot，Word2Vec 更像是给词一个“语义坐标”。

但它的问题是：一个词通常只有一个固定向量。

例如：

```text
我吃了一个苹果。
我买了一台苹果手机。
```

Word2Vec 很难仅靠固定词向量区分两个“苹果”的不同含义。

### 4.4 ELMo

ELMo 的关键升级是上下文相关表示。

也就是说，同一个词在不同句子里可以有不同向量。

```text
我吃了一个苹果。      -> 苹果偏水果
我买了一台苹果手机。  -> 苹果偏品牌
```

这说明文本表示从“静态词向量”开始走向“动态上下文表示”。

## 5. 本章核心总结

文本表示的发展可以这样概括：

```text
VSM / One-hot：有词就行，但稀疏、不懂语义
N-gram：开始看局部词序，也能做早期语言模型
Word2Vec：开始学习词的语义距离，但词向量固定
ELMo：开始根据上下文动态调整词表示
Transformer / LLM：进一步用注意力机制建模上下文中的 token 表示
```

最重要的一句话：

```text
文本表示经历了从稀疏、固定、局部，逐步发展到稠密、语义化、上下文相关。Transformer 和 LLM 是这条路线的后续。
```

## 6. 我已经掌握的点

- NLP 的目标是让计算机理解、处理和生成自然语言。
- 现代 LLM 通常通过 tokenizer 把文本切成 token，而不是直接处理完整词。
- 神经网络只能处理数字，所以文本必须经过数字化和向量化。
- One-hot / VSM 的主要问题是高维稀疏、不懂词序、不懂语义。
- N-gram 既可以做文本特征，也可以做统计语言模型，还可以用于生成结果评价。
- N-gram 和 GPT 都有“根据前文预测后文”的思想，但 GPT 使用神经网络和更长上下文。
- Word2Vec 是静态词向量，ELMo 是上下文相关的动态词向量。
