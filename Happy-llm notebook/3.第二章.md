# Happy-LLM 第二章：Transformer 架构笔记

## 本章定位

第二章的核心不是背 Transformer 图，而是理解一句话：

```text
Transformer 用 Attention 让 token 之间并行交流，再用 FFN 加工每个 token 自己的表示，最后通过多层堆叠得到上下文语义。
```

本章最重要的链路是：

```text
token id
-> token embedding
-> positional encoding
-> self-attention
-> multi-head attention
-> layer norm + residual
-> FFN
-> encoder / decoder
-> lm_head
```

## 0. 学习检测结果

本章小测已通过。

已经掌握的点：

- Transformer 需要位置编码，因为 Attention 本身不天然包含顺序信息。
- Q、K、V 可以理解为查询、被匹配的信息索引、最终被加权汇总的信息内容。
- 多头注意力是从不同表示子空间观察 token 关系，最后 concat 后再融合。
- LayerNorm 对最后一维 hidden dim 做归一化。
- EncoderLayer 是 Self-Attention + FFN。
- DecoderLayer 是 Masked Self-Attention + Cross-Attention + FFN。
- Pre-LN 结构是每个子层前 LN，每个子层后残差连接。

需要修正的点：

```text
1. causal mask 用的是 -inf，不是 +inf。softmax(-inf) = 0。
2. FFN 不是数据增强，而是对每个 token 的 hidden 向量做非线性特征加工。
3. Transformer 的并行主要指训练时可以一次性计算所有位置；自回归生成时仍然要逐 token 生成。
```

## 1. 什么叫并行计算

RNN / LSTM 的计算是顺序依赖的：

```text
h1 依赖 h0
h2 依赖 h1
h3 依赖 h2
```

所以第 3 个位置必须等前面算完。

Transformer 的 self-attention 不按时间一步步递推，而是一次性把所有 token 的 Q、K、V 都算出来：

```text
x: [B, T, C]
Q/K/V: [B, T, C]
attention scores: [B, H, T, T]
```

其中 `[T, T]` 表示每个 token 对每个 token 的注意力分数。这个矩阵可以通过一次矩阵乘法算出，所以训练时更容易并行。

但是 Decoder 生成时不能完全并行，因为未来 token 还不存在：

```text
先生成第 1 个 token
再用第 1 个生成第 2 个
再用前 2 个生成第 3 个
```

所以要区分：

```text
训练时：目标序列已知，可以并行算所有位置，但用 causal mask 防止偷看未来。
推理时：目标序列未知，需要一个 token 一个 token 生成。
```

## 2. 位置编码

Attention 本身只看 token 之间的相似度，如果没有位置编码，模型不容易区分：

```text
我 喜欢 你
你 喜欢 我
```

所以 Transformer 要把位置信息加到 token embedding 上：

```text
最终输入 = token embedding + positional encoding
```

形状不变：

```text
token embedding:      [B, T, C]
positional encoding:  [1, T, C]
相加后:               [B, T, C]
```

正余弦位置编码里，sin 和 cos 是成对出现的：

```text
第 0 列：sin，i = 0
第 1 列：cos，i = 0
第 2 列：sin，i = 1
第 3 列：cos，i = 1
```

每一对 sin/cos 共享一个频率。

## 3. Q、K、V 的理解

Self-Attention 里，每个 token 都会变成 Q、K、V 三个向量。

可以这样理解：

```text
Q：当前 token 发出的查询，我想找什么信息？
K：每个 token 提供的匹配标签，我适合被谁关注？
V：每个 token 真正贡献的内容，被注意力权重加权汇总。
```

计算流程：

```text
Q @ K.T -> 得到注意力分数
softmax -> 得到注意力权重
权重 @ V -> 得到融合上下文后的 token 表示
```

缩放项：

```text
Q @ K.T / sqrt(d_k)
```

`/ sqrt(d_k)` 是为了避免维度大时点积值过大，导致 softmax 过于尖锐、梯度不稳定。

## 4. causal mask

Decoder 的 masked self-attention 不能看未来 token。

mask 通常是上三角 `-inf`：

```text
[[0, -inf, -inf, -inf],
 [0,    0, -inf, -inf],
 [0,    0,    0, -inf],
 [0,    0,    0,    0]]
```

它会加到 attention scores 上：

```text
scores = scores + mask
```

未来位置被加上 `-inf` 后：

```text
softmax(-inf) = 0
```

所以模型在当前位置不会关注未来 token。

一句话：

```text
causal mask 用 -inf 遮住未来位置，让训练时可以并行计算，但信息上不能偷看未来。
```

## 5. 多头注意力

如果 `C = 512`，`H = 8`，那么每个 head 的维度是：

```text
D = C / H = 64
```

形状变化：

```text
x:         [B, T, C]
Q/K/V:     [B, T, C]
拆多头:    [B, T, H, D]
转置:      [B, H, T, D]
scores:    [B, H, T, T]
乘 V:      [B, H, T, D]
转回:      [B, T, H, D]
concat:    [B, T, C]
输出投影:  [B, T, C]
```

多头的意义是：

```text
不同 head 可以从不同表示子空间关注不同关系。
```

例如某些 head 可能关注相邻词，某些 head 可能关注主谓关系，某些 head 可能关注长距离依赖。

最后 concat 是为了先保留每个 head 的不同视角，再通过输出线性层 `W_o` 统一融合。

## 6. LayerNorm 和残差连接

Transformer 常用 LayerNorm。若输入：

```text
x: [B, T, C]
```

则：

```python
mean = x.mean(-1, keepdim=True)
```

形状是：

```text
mean: [B, T, 1]
```

LayerNorm 对每个 token 自己的 hidden 维度 `C` 做归一化。

Pre-LN Transformer 的结构是：

```text
x = x + Attention(LN(x))
x = x + FFN(LN(x))
```

对于 DecoderLayer：

```text
x = x + MaskedSelfAttention(LN(x))
x = x + CrossAttention(LN(x), enc_out, enc_out)
x = x + FFN(LN(x))
```

残差连接的作用：

```text
保留原始信息，让子层只学习需要补充或修改的部分，也让深层网络更容易训练。
```

## 7. FFN / MLP 的作用

Attention 负责 token 之间的信息交流：

```text
每个 token 看其他 token
```

FFN 负责每个 token 自己内部的非线性加工：

```text
每个 token 的 hidden 向量自己变换
```

常见结构：

```text
[B, T, C]
-> Linear1
[B, T, 4C]
-> Activation
[B, T, 4C]
-> Linear2
[B, T, C]
```

所以 FFN 不是数据增强，也不是直接“总体概括”。它更准确的作用是：

```text
在 Attention 融合上下文后，对每个 token 的表示做非线性特征加工，提高表达能力。
```

## 8. Encoder 和 Decoder

EncoderLayer：

```text
Self-Attention，不带 causal mask
FFN
```

Encoder 可以看完整输入序列，因为它负责理解源序列，不是在生成。

DecoderLayer：

```text
Masked Self-Attention，不能看未来目标 token
Cross-Attention，读取 Encoder 输出
FFN
```

Cross-Attention 里：

```text
Q 来自 Decoder 当前状态
K 来自 Encoder 输出
V 来自 Encoder 输出
```

它的意义是：

```text
Decoder 在生成目标 token 时，去查看 Encoder 对源序列的理解。
```

例如机器翻译：

```text
源序列：I like you
目标序列：我 喜欢 你
```

生成“喜欢”时，Decoder 可以通过 cross-attention 关注 Encoder 输出中和 `like` 相关的位置。

## 9. 整体 Transformer 代码链路

完整模型大致是：

```text
idx: [B, T]
-> token embedding: [B, T, C]
-> positional encoding: [B, T, C]
-> dropout: [B, T, C]
-> encoder: [B, T, C]
-> decoder: [B, T, C]
-> lm_head: [B, T, vocab_size]
```

训练时：

```text
logits:  [B, T, vocab_size]
targets: [B, T]
```

拉平成：

```text
logits:  [B*T, vocab_size]
targets: [B*T]
```

然后用 cross entropy 计算每个位置预测 token 的损失。

推理时通常只取最后一个位置：

```text
x[:, [-1], :] -> [B, 1, C]
logits -> [B, 1, vocab_size]
```

因为自回归生成只需要预测下一个 token。

## 10. 代码实现细节补充

这一节记录第三遍看 Transformer 实现时容易漏掉的代码细节。

### 10.1 多头注意力为什么要把 head 放到第二维

多头注意力输入通常是：

```text
x: [B, T, C]
```

其中：

```text
B = batch size
T = seq_len
C = hidden dim
H = n_heads
D = head_dim = C / H
```

代码先用一个大的线性层一次性算出所有 head 的 Q/K/V：

```python
self.wq = nn.Linear(args.n_embd, self.n_heads * self.head_dim, bias=False)
self.wk = nn.Linear(args.n_embd, self.n_heads * self.head_dim, bias=False)
self.wv = nn.Linear(args.n_embd, self.n_heads * self.head_dim, bias=False)

xq, xk, xv = self.wq(q), self.wk(k), self.wv(v)
```

形状还是：

```text
[B, T, C]
```

但这时的 `C` 已经可以被理解成：

```text
H 个 head，每个 head D 维
```

所以接下来拆成多头：

```python
xq = xq.view(bsz, seqlen, self.n_heads, self.head_dim)
xk = xk.view(bsz, seqlen, self.n_heads, self.head_dim)
xv = xv.view(bsz, seqlen, self.n_heads, self.head_dim)
```

形状变成：

```text
[B, T, H, D]
```

然后转置：

```python
xq = xq.transpose(1, 2)
xk = xk.transpose(1, 2)
xv = xv.transpose(1, 2)
```

形状变成：

```text
[B, H, T, D]
```

这样做的原因是：PyTorch 的 `matmul` 默认拿最后两个维度做矩阵乘法，前面的维度当作批量维度。

所以：

```python
scores = torch.matmul(xq, xk.transpose(2, 3)) / math.sqrt(self.head_dim)
```

对应：

```text
xq:              [B, H, T, D]
xk.transpose:    [B, H, D, T]
scores:          [B, H, T, T]
```

这里的 `H` 放在第二维，就等于让每个 head 独立拥有一个 `[T, T]` 的注意力矩阵。

如果不先变成 `[B, H, T, D]`，就不方便让每个 head 各自独立计算 token 之间的注意力。

一句话：

```text
把 n_heads 放到第二维，是为了让 matmul 在每个 batch、每个 head 内部独立计算 [T, T] 注意力矩阵。
```

### 10.2 view、transpose、contiguous 的注意点

拆多头时：

```python
xq = xq.view(bsz, seqlen, self.n_heads, self.head_dim)
xq = xq.transpose(1, 2)
```

这里不能直接把 `[B, T, C]` view 成 `[B, H, T, D]`，因为 `view` 是按底层连续内存顺序重新解释张量。

正确做法是：

```text
先 [B, T, C] -> [B, T, H, D]
再交换 T 和 H -> [B, H, T, D]
```

合并多头时：

```python
output = output.transpose(1, 2).contiguous().view(bsz, seqlen, -1)
```

含义是：

```text
[B, H, T, D]
-> transpose
[B, T, H, D]
-> view
[B, T, H*D] = [B, T, C]
```

`contiguous()` 的作用是重新整理内存。因为 `transpose` 只改变张量视图，不一定改变底层内存排列，直接 `view` 可能报错。

一句话：

```text
transpose 后如果要 view，通常先 contiguous。
```

### 10.3 causal mask：并行计算但不能偷看未来

代码：

```python
mask = torch.full((1, 1, args.max_seq_len, args.max_seq_len), float("-inf"))
mask = torch.triu(mask, diagonal=1)
self.register_buffer("mask", mask)
```

mask 形状是：

```text
[1, 1, max_seq_len, max_seq_len]
```

它会广播到：

```text
scores: [B, H, T, T]
mask:   [1, 1, T, T]
```

上三角是未来位置：

```text
[[0, -inf, -inf, -inf],
 [0,    0, -inf, -inf],
 [0,    0,    0, -inf],
 [0,    0,    0,    0]]
```

加到分数上：

```python
scores = scores + self.mask[:, :, :seqlen, :seqlen]
scores = F.softmax(scores.float(), dim=-1).type_as(xq)
```

因为：

```text
softmax(-inf) = 0
```

所以未来 token 的注意力权重会变成 0。

注意这里的重点是：

```text
训练时可以并行算所有位置的 scores，但 mask 保证信息上不能看未来。
```

推理生成时仍然需要逐 token 生成，因为未来 token 还不存在。

### 10.4 Decoder 的两段注意力到底分别看什么

DecoderLayer 代码：

```python
norm_x = self.attention_norm_1(x)
x = x + self.mask_attention.forward(norm_x, norm_x, norm_x)

norm_x = self.attention_norm_2(x)
h = x + self.attention.forward(norm_x, enc_out, enc_out)

out = h + self.feed_forward.forward(self.ffn_norm(h))
```

第一段是 masked self-attention：

```text
Q = Decoder 当前状态
K = Decoder 当前状态
V = Decoder 当前状态
```

作用：

```text
看目标端已经生成的前文。
```

它类似 RNN 中“根据前文建立当前状态”的作用，但实现方式不同：

```text
RNN：一步一步递推。
Masked Self-Attention：矩阵并行计算 + causal mask 禁止看未来。
```

第二段是 cross-attention：

```text
Q = Decoder 当前状态
K = Encoder 输出 enc_out
V = Encoder 输出 enc_out
```

作用：

```text
生成目标 token 时，回头查源句子的语义记忆。
```

这不是把英文原文混进中文输出，而是用 Decoder 的查询去读取 Encoder 编码后的语义向量。

可以这样类比：

```text
Q：我现在中文生成到这里了，需要查什么？
K：源句子每个位置的索引/匹配信息。
V：源句子每个位置的语义内容。
```

所以 `V` 必须是 `enc_out`。因为注意力权重算出来后，真正被加权汇总的是 `V`：

```python
output = torch.matmul(scores, xv)
```

如果 cross-attention 的 `V` 不来自 Encoder，Decoder 就无法把源句子的语义内容取回来。

一句话：

```text
Decoder 第一段解决“我已经生成了什么”，第二段解决“原文到底说了什么”。
```

### 10.5 Encoder 和 Decoder 一起工作时的状态

机器翻译视角：

```text
源句子：I like you
目标句子：我 喜欢 你
```

Encoder 做：

```text
读取完整源句子
-> 每个源 token 得到上下文表示
-> enc_out: [B, src_len, C]
```

Decoder 做：

```text
读取目标端前文
-> masked self-attention 建立目标端上下文
-> cross-attention 查询 enc_out
-> FFN 加工当前 token 表示
```

Cross-attention 中如果源序列长度和目标序列长度不同，理论形状是：

```text
Q: [B, H, tgt_len, D]
K: [B, H, src_len, D]
V: [B, H, src_len, D]
```

注意力分数：

```text
Q @ K^T -> [B, H, tgt_len, src_len]
```

乘 V：

```text
[B, H, tgt_len, src_len] x [B, H, src_len, D]
-> [B, H, tgt_len, D]
```

最后合并多头：

```text
[B, tgt_len, C]
```

这说明：

```text
每个目标 token 都可以关注源句子的所有位置。
```

### 10.6 LayerNorm 里的 a_2 和 b_2

代码：

```python
self.a_2 = nn.Parameter(torch.ones(features))
self.b_2 = nn.Parameter(torch.zeros(features))
```

它们是 LayerNorm 的可学习缩放和平移参数：

```text
a_2 = gamma / weight
b_2 = beta / bias
```

如果：

```text
x: [B, T, C]
features = C
```

那么：

```text
a_2: [C]
b_2: [C]
```

forward：

```python
mean = x.mean(-1, keepdim=True)
std = x.std(-1, keepdim=True)
return self.a_2 * (x - mean) / (std + self.eps) + self.b_2
```

形状：

```text
mean: [B, T, 1]
std:  [B, T, 1]
a_2:  [C] -> broadcast -> [B, T, C]
b_2:  [C] -> broadcast -> [B, T, C]
```

意义：

```text
先标准化，让数值稳定；
再让模型自己学习每个 hidden 维度要放大、缩小、上移或下移。
```

### 10.7 FFN 在代码框架里的位置

FFN 不负责 token 之间通信，token 之间通信已经由 Attention 完成。

FFN 的作用是：

```text
对每个 token 自己的 hidden 向量做非线性加工。
```

常见形状：

```text
[B, T, C]
-> Linear1
[B, T, 4C]
-> Activation
[B, T, 4C]
-> Linear2
[B, T, C]
```

在 DecoderLayer 里：

```python
out = h + self.feed_forward.forward(self.ffn_norm(h))
```

这里的 `h` 是 FFN 子层的残差输入：

```text
FFN 输出的是“对 h 的补充加工”
out = 原 h + 加工结果
```

一句话：

```text
Attention 负责看别人，FFN 负责加工自己，残差负责保留原信息。
```

### 10.8 target_len、训练梯度和推理生成

`target_len` 不是模型自己猜出来的，而是由传进 Decoder 的目标端输入长度决定的：

```python
target_len = x.size(1)
```

如果：

```text
x: [B, target_len, C]
```

那么 Decoder 的 masked self-attention 会一次性处理这 `target_len` 个位置。

#### 训练时：整段目标序列已知

翻译训练中，假设目标句子是：

```text
<BOS> 我 喜欢 你
```

训练样本通常构造成：

```text
idx:     <BOS> 我 喜欢
targets: 我    喜欢 你
```

所以训练时 Decoder 输入的是所有已知目标端前缀：

```text
idx: [B, T]
```

模型一次性输出所有位置的预测：

```text
logits: [B, T, vocab_size]
```

每个位置都预测下一个 token：

```text
位置0：根据 <BOS>          预测 我
位置1：根据 <BOS> 我       预测 喜欢
位置2：根据 <BOS> 我 喜欢  预测 你
```

代码中通常把所有位置拉平后算交叉熵：

```python
loss = F.cross_entropy(
    logits.view(-1, logits.size(-1)),  # [B*T, vocab_size]
    targets.view(-1),                  # [B*T]
    ignore_index=-1
)
```

这表示：

```text
所有 batch、所有时间位置都参与同一个平均 loss。
```

然后：

```python
loss.backward()
optimizer.step()
```

PyTorch 会从 loss 自动反向传播，给所有参与计算的参数算梯度：

```text
Embedding
Wq / Wk / Wv / Wo
FFN 里的 Linear
LayerNorm 的 a_2 / b_2
lm_head
```

所以训练时是：

```text
计算上并行：一次性算所有位置。
信息上自回归：causal mask 禁止每个位置看未来。
```

#### 推理时：普通做法传入已有全部 token

推理时没有 `targets`，也不算 loss 和梯度。

普通教学代码一般这样生成：

```text
第1步输入：<BOS>
第2步输入：<BOS> 我
第3步输入：<BOS> 我 喜欢
```

每一步都把当前已经生成的全部 token 重新传入模型：

```python
logits = model(idx)              # [B, T, vocab_size]
next_logits = logits[:, -1, :]   # [B, vocab_size]
next_id = sample_or_argmax(next_logits)
idx = torch.cat([idx, next_id], dim=1)
```

虽然模型会重新算所有位置的 logits：

```text
logits: [B, T, vocab_size]
```

但推理时只取最后一个位置：

```text
logits[:, -1, :]
```

因为最后一个位置代表：

```text
根据当前完整前文，预测下一个 token。
```

普通推理的问题是：

```text
每一步都会重复计算前面 token 的 attention，简单但慢。
```

#### 高效推理：KV Cache 只算最新 token

实际大模型推理通常使用 KV Cache。

这时每一步只用最新 token 生成新的 Q，同时复用过去已经算好的 K/V：

```text
Q: 当前最新 token
K/V: 历史缓存 + 当前 token 的 K/V
```

self-attention 形状变成：

```text
Q:      [B, H, 1, D]
K/V:    [B, H, current_len, D]
scores: [B, H, 1, current_len]
```

cross-attention 中，Encoder 输出可以提前算好并复用：

```text
Q:      [B, H, 1, D]
K/V:    [B, H, src_len, D]
scores: [B, H, 1, src_len]
```

所以推理时你的直觉是对的：

```text
新一步生成时，真正需要新算的是最新 token 对历史前文和 Encoder 输出的注意力。
```

一句话总结：

```text
训练时：整段目标序列已知，一次性算所有位置的 logits 和 loss，再反向传播算梯度。
普通推理：每次传入已有全部 token，重新算全部位置，但只取最后一个位置预测。
高效推理：用 KV Cache 复用历史 K/V，只为最新 token 计算 attention。
```
