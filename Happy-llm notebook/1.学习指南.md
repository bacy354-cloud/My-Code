# Happy-LLM 学习指南

## 总体认知

Happy-LLM 是一门“从零开始理解并搭建大模型”的系统课。它不像 LLM-Algo-LeetCode 那样偏刷题和底层算法面试，而是按 LLM 的完整知识地图展开：NLP 基础、Transformer、预训练语言模型、大语言模型、手写模型、训练流程、RAG 和 Agent 应用。

这门课最适合帮你建立全局感：LLM 到底由哪些部分组成，每个部分解决什么问题，代码和理论分别在哪里落地。

你的学习目标不应该是“把所有章节看完”，而是逐步形成四种能力：

- 能说清 LLM 从 NLP 到 Transformer 再到大模型的演化逻辑。
- 能读懂并改动一个小型 Transformer / LLaMA2 风格模型。
- 能理解预训练、SFT、PEFT、LoRA、QLoRA 的区别。
- 能做一个小型 RAG 或 Agent demo，把模型应用串起来。

本机是 CPU-only，因此策略是：本机完成阅读、代码导读、小模型结构、RAG/Agent 小 demo；大规模预训练、DeepSpeed、多卡训练放云 GPU。

## 知识链路

Happy-LLM 的知识链路可以这样看：

```text
NLP 基础
  -> 文本表示：词、token、embedding
  -> Transformer：Attention / FFN / LayerNorm / Residual
  -> PLM：BERT / T5 / GPT 架构分化
  -> LLM：Decoder-only / Scaling / Emergent Ability
  -> 手写 LLaMA2：Tokenizer / Model / Dataset / Training Loop
  -> 训练实践：Pretrain / SFT / PEFT / LoRA / QLoRA
  -> 应用层：Evaluation / RAG / Agent
```

每章学习时，优先抓“这个东西解决什么问题”，再看结构和代码。比如 Attention 解决长距离依赖和信息聚合；Tokenizer 解决文本到离散 token 的映射；LoRA 解决全量微调成本过高；RAG 解决模型参数记忆不够新、不够可控的问题。

## 阶段式学习路径

### 阶段一：建立 LLM 全景认知

范围：学习与环境准备、前言、第一章、第四章粗读。

目标：先知道 LLM 世界有哪些模块，不要一上来陷入公式细节。

重点问题：

- NLP 任务和 LLM 任务有什么关系？
- 为什么大模型主流是 Decoder-only？
- 预训练、微调、对齐、应用分别处在哪个层次？
- 本机 CPU 能做什么，不能做什么？

最小成果物：

- 一页笔记：`LLM 从输入文本到输出答案经过哪些环节`。
- 一张图或列表：`NLP -> Transformer -> PLM -> LLM -> RAG/Agent`。

### 阶段二：Transformer 主干结构

范围：第二章 Transformer 架构 + `docs/chapter2/code/transformer.py`

目标：把 Transformer 拆成可理解的模块，而不是背一整张大图。

学习顺序：

1. Token 和 Embedding。
2. Attention：Q、K、V 是什么，为什么要 softmax。
3. Multi-Head Attention：为什么要多头。
4. FFN：为什么 Attention 后还要 MLP。
5. Residual 和 LayerNorm：为什么深层网络需要稳定训练。
6. Encoder-Decoder 和 Decoder-only 的区别。

最小成果物：

- 能 import `transformer.py`。
- 能手写一段说明：`Attention 的输入输出 shape`。
- 能和 LLM-Algo 的 RoPE、MHA/GQA notebook 对上。

已验证：

```text
chapter2 transformer import ok
```

### 阶段三：预训练语言模型到 LLM

范围：第三章、第四章。

目标：理解 BERT、T5、GPT 不是孤立模型，而是三种架构路线。

重点：

- Encoder-only：擅长理解类任务。
- Encoder-Decoder：擅长输入输出都比较复杂的生成任务。
- Decoder-only：适合统一成 next token prediction，是当前 LLM 主线。
- Scaling Law、涌现能力、上下文学习只是现象，底层仍是 token 预测和大规模训练。

最小成果物：

- 一张对比表：`BERT / T5 / GPT 的输入、输出、训练目标、适用任务`。
- 一段解释：`为什么 GPT 类模型适合做通用对话助手`。

### 阶段四：动手搭建一个小型 LLM

范围：第五章 + `docs/chapter5/code`

目标：从代码上看清楚一个小型 LLM 项目需要哪些文件。

重点文件：

- `k_model.py`：模型结构。
- `dataset.py`：数据读取与样本构造。
- `train_tokenizer.py`：Tokenizer 训练。
- `ddp_pretrain.py` / `ddp_sft_full.py`：训练脚本，先读懂，不在本机硬跑。
- `model_sample.py`：采样生成。

最小成果物：

- 能 import `k_model.py` 和 `dataset.py`。
- 能画出：`文本 -> tokenizer -> token ids -> model -> logits -> loss/generation`。
- 能解释训练脚本里模型、数据、优化器、loss 分别在哪里。

已验证：

```text
chapter5 model/dataset import ok
```

本机边界：可以读代码和小样本运行；不要下载大数据集或训练大模型。

### 阶段五：训练流程、PEFT 和 LoRA

范围：第六章。

目标：理解现代 LLM 训练流程，但不把本机当训练服务器。

学习顺序：

1. Pretrain：让模型学语言统计规律。
2. SFT：让模型学指令格式和任务输出。
3. PEFT：只训练少量参数，降低成本。
4. LoRA/QLoRA：低秩适配和低比特加载。
5. DeepSpeed/ZeRO：解决显存和分布式训练问题。

最小成果物：

- 一页笔记：`Pretrain / SFT / LoRA / QLoRA / DPO 分别改了什么`。
- 一份实验模板：模型名、数据集、训练命令、随机种子、显存、结果、失败原因。

本机边界：本机只做代码阅读和小规模 import；DeepSpeed、多卡、正式微调放云 GPU。

### 阶段六：RAG、Agent 和应用 demo

范围：第七章 + `docs/chapter7/RAG` + `docs/chapter7/Agent`

目标：把 LLM 从“模型本体”扩展到“可用应用”。

RAG 链路：

```text
文档
  -> 清洗和切块
  -> embedding
  -> 向量库
  -> 检索 top-k
  -> 拼接 prompt
  -> LLM 生成回答
```

Agent 链路：

```text
任务
  -> 规划
  -> 调用工具
  -> 观察结果
  -> 继续决策
  -> 输出答案
```

最小成果物：

- 一个本地小 RAG demo：只用 2-3 篇短文本，不接大数据集。
- 一页笔记：`RAG 解决了什么，Agent 又多了什么`。

已验证：

```text
chapter7 Agent demo import ok
chapter7 RAG modules import ok
```

## 学习节奏建议

你时间不固定，所以不要按天排。建议按“阶段完成度”推进：

- 每次只选一个阶段里的一个小目标。
- 每次学习结束必须留下最小成果物：一页笔记、一次 import 通过、一个 notebook 测试通过、一个小 demo。
- 如果连续卡住超过 30 分钟，先写下卡点，再切到阅读或笔记，不要硬耗。
- 专业课忙的时候，只保留阅读和小笔记；空闲时再补代码实验。
