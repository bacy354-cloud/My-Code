## 本节目标

理解并实现 RoPE，也就是旋转位置编码。

本节重点：

1. RoPE 不是给 `x` 加位置向量，而是旋转 `q/k`。
2. `freqs_cis` 是一张按位置和特征组生成的复数旋转表。
3. `head_dim` 最后一维要两两配对，变成复数。
4. 旋转表只管 `seq` 和特征组，不管 `batch/head`，这两个维度靠广播复制。
5. `view_as_complex` / `view_as_real` 的 shape 变化要能说清楚。

## 核心直觉

Attention 分数来自：

```python
score = q @ k.T
```

所以 RoPE 把位置信息塞进 `q/k`，而不是直接加到原始 `x` 上。

可以记成：

```text
RoPE = 把 q/k 按 token 位置旋转，让 attention score 自然带上相对距离。
```

如果 q 在第 `m` 个位置，k 在第 `n` 个位置：

```text
q 旋转 m * theta
k 旋转 n * theta
```

两者做内积时，旋转角度会变成差：

```text
m * theta - n * theta = (m - n) * theta
```

所以 RoPE 用绝对位置旋转，最后在 attention 里得到相对位置信息。

## 两两配对

RoPE 的旋转发生在二维平面里，所以要把最后一维两两配对。

例如：

```text
head_dim = 64

[x0, x1, x2, x3, ..., x62, x63]
```

配成：

```text
[x0, x1]    第 1 个二维特征组
[x2, x3]    第 2 个二维特征组
...
[x62, x63]  第 32 个二维特征组
```

所以：

```text
64 个实数 -> 32 组 -> 每组 2 个实数
```

shape 链：

```text
(batch, seq, heads, head_dim)
-> (batch, seq, heads, head_dim/2, 2)
-> (batch, seq, heads, head_dim/2)  # 复数
```

关键点：

```text
只要是 complex tensor，实部/虚部那个 2 就藏进 dtype 里，不再显示在 shape 里。
```

## TODO1：生成旋转表

函数：

```python
def precompute_freqs_cis(dim: int, end: int, theta: float = 10000.0):
```

参数含义：

```text
dim = head_dim，管每个 token 里有多少特征
end = 要预计算多少个位置，通常等于 seq_len
theta = 生成一串频率用的基数
```

先记口诀：

```text
位置用 end。
特征用 dim。
旋转表 = 位置 outer 特征组频率。
```

代码：

```python
freqs = 1.0 / (theta ** (torch.arange(0, dim, 2).float() / dim))
t = torch.arange(end, device=freqs.device, dtype=torch.float32)
freqs = torch.outer(t, freqs)
freqs_cis = torch.polar(torch.ones_like(freqs), freqs)
return freqs_cis
```

逐步理解：

```python
torch.arange(0, dim, 2)
```

表示只取偶数 index，因为最后一维要两两配对。

如果：

```python
dim = 8
```

那么：

```text
[0, 2, 4, 6]
```

最终得到 `dim/2` 个特征组频率。

接着：

```python
t = torch.arange(end)
```

表示 token 位置编号：

```text
[0, 1, 2, ..., end-1]
```

然后：

```python
freqs = torch.outer(t, freqs)
```

外积的意思是：

```text
每个 token 位置编号 * 每个特征组频率
```

得到的是旋转角度表，不再是单纯频率。

最后：

```python
torch.polar(torch.ones_like(freqs), freqs)
```

含义：

```text
半径 = 全 1，形状照着 freqs 来
角度 = freqs 这张角度表
```

半径全 1 是因为 RoPE 只旋转，不拉伸。

## 广播旋转表

TODO1 得到：

```text
freqs_cis: (16, 32)
```

TODO2 得到：

```text
xq_: (2, 16, 4, 32)
```

`freqs_cis` 只管：

```text
seq 位置
特征组
```

不管：

```text
batch
head
```

所以要 reshape 成：

```text
(1, 16, 1, 32)
```

这样可以广播到：

```text
(2, 16, 4, 32)
```

代码：

```python
def reshape_for_broadcast(freqs_cis: torch.Tensor, x: torch.Tensor):
    ndim = x.ndim
    shape = [d if i == 1 or i == ndim - 1 else 1 for i, d in enumerate(x.shape)]
    return freqs_cis.view(*shape)
```

等价的好懂写法：

```python
shape = []
for i, d in enumerate(x.shape):
    if i == 1 or i == x.ndim - 1:
        shape.append(d)
    else:
        shape.append(1)
```

含义：

```text
第 1 维 seq 保留。
最后一维特征组保留。
其他维度变成 1，用来广播。
```

一句话：

```text
RoPE 对 token 位置不复制，因为不同位置要不同旋转；
RoPE 只在 batch 和 head 上复制同一套旋转规则。
```

## TODO3：复数相乘并转回实数

先广播：

```python

```

然后复数相乘：

```python
xq_ * freqs_cis
```

含义：

```text
每个 token、每个 head 里的每个复数特征组，
乘上由 token 位置编号和特征组频率生成的旋转因子。
```

shape 不变：

```text
(2, 16, 4, 32)
```

再拆回实数：

```python

```

```text
(2, 16, 4, 32)

```

```python

```

```python

```

## 最终代码

```python
import torch

def precompute_freqs_cis(dim: int, end: int, theta: float = 10000.0):
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2).float() / dim))
    t = torch.arange(end, device=freqs.device, dtype=torch.float32)
    freqs = torch.outer(t, freqs)
    freqs_cis = torch.polar(torch.ones_like(freqs), freqs)
    return freqs_cis

def reshape_for_broadcast(freqs_cis: torch.Tensor, x: torch.Tensor):
    ndim = x.ndim
    shape = [d if i == 1 or i == ndim - 1 else 1 for i, d in enumerate(x.shape)]
    return freqs_cis.view(*shape)

def apply_rotary_emb(
    xq: torch.Tensor,
    xk: torch.Tensor,
    freqs_cis: torch.Tensor,
) -> tuple[torch.Tensor, torch.Tensor]:
    xq_ = torch.view_as_complex(xq.float().reshape(*xq.shape[:-1], -1, 2))
    xk_ = torch.view_as_complex(xk.float().reshape(*xk.shape[:-1], -1, 2))

    freqs_cis = reshape_for_broadcast(freqs_cis, xq_)

    xq_out = torch.view_as_real(xq_ * freqs_cis).flatten(3)
    xk_out = torch.view_as_real(xk_ * freqs_cis).flatten(3)

    return xq_out.type_as(xq), xk_out.type_as(xk)
```

## Shape 总链路

以：

```text
xq: (2, 16, 4, 64)
```

为例：

```text
生成旋转表：

特征组频率:      (32,)
位置编号:        (16,)
outer 后角度表:  (16, 32)
polar 后旋转表:  (16, 32)  # 复数

应用 RoPE：

xq 实数:          (2, 16, 4, 64)
两两配对:         (2, 16, 4, 32, 2)
转成复数:         (2, 16, 4, 32)
旋转表广播:       (1, 16, 1, 32)
复数相乘:         (2, 16, 4, 32)
拆回复数的实部虚部: (2, 16, 4, 32, 2)
压回 head_dim:    (2, 16, 4, 64)
```

## 易错点

1. `end` 是位置数量，通常等于 `seq_len`；`dim` 是 `head_dim`。
2. `t = torch.arange(end)`，不要写成 `torch.arange(dim)`。
3. `torch.outer(t, freqs)` 的顺序不要反，目标 shape 是 `(end, dim//2)`。
4. `outer` 后得到的是旋转角度表，不是频率表。
5. `torch.ones_like(freqs)` 是为了生成同形状的全 1 半径表。
6. `view_as_complex` 后，最后的 `2` 会藏进复数 dtype，不再显示在 shape 里。
7. `freqs_cis` 要先 reshape 成 `(1, seq, 1, dim/2)`，再和 `xq_` 相乘。
8. `view_as_real` 是把复数拆回两个实数，不是放大。
9. `flatten(3)` 是从第 3 维开始合并，把 `(dim/2, 2)` 压回 `dim`。
