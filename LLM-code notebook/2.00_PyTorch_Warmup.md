## Part 1：Tensor 维度变换

输入图像特征：

```text
x: (batch, channels, height, width)
```

Transformer 需要序列形式：

```text
output: (batch, height * width, channels)
```

原生 PyTorch 写法：

```python
batch, channels, height, width = x.shape
x_native = x.permute(0, 2, 3, 1).reshape(batch, height * width, channels)
```

`einops` 写法：

```python
from einops import rearrange

x_einops = rearrange(x, "b c h w -> b (h w) c")
```

核心理解：

```text
permute：换维度顺序
reshape：合并或拆分维度
(h w)：把 height 和 width 合并成一个序列维度
```

例子：

```text
(8, 64, 14, 14) -> (8, 196, 64)
```

## Part 2：Embedding 的本质

Embedding 本质是查表。

```text
input_ids: (batch, seq)
weight: (vocab_size, hidden_dim)
output: (batch, seq, hidden_dim)
```

官方写法：

```python
emb_layer = nn.Embedding(vocab_size, hidden_dim)
out_official = emb_layer(input_ids)
```

手动查表：

```python
out_manual = emb_layer.weight[input_ids]
```

核心理解：

```text
input_ids 里每个数字，都是去 weight 中取一行。
input_ids 是几维，输出前面就保留几维，最后补一个 hidden_dim。
```

例子：

```text
input_ids: (2, 5)
weight: (100, 32)
output: (2, 5, 32)
```

## Part 3：Linear + ReLU 自定义 Autograd

前向：

```python
z = x @ weight.T + bias
y = F.relu(z)
mask = (z > 0).float()
ctx.save_for_backward(x, weight, mask)
```

这里的 `mask` 不只是普通遮罩，它本质上是 ReLU 的局部导数：

```text
z > 0  -> dy/dz = 1
z <= 0 -> dy/dz = 0
```

反向：

```python
x, weight, mask = ctx.saved_tensors

grad_z = grad_output * mask

grad_x = grad_z @ weight
grad_weight = grad_z.T @ x
grad_bias = grad_z.sum(dim=0)

return grad_x, grad_weight, grad_bias
```

## 形状链条

设：

```text
x:      (4, 3)
weight: (5, 3)
bias:   (5,)
```

前向：

```text
z = x @ weight.T + bias -> (4, 5)
y = relu(z)             -> (4, 5)
mask                    -> (4, 5)
```

反向：

```text
grad_output 对应 y -> (4, 5)
grad_z 对应 z      -> (4, 5)

grad_x      = grad_z @ weight  -> (4, 3)
grad_weight = grad_z.T @ x     -> (5, 3)
grad_bias   = grad_z.sum(dim=0)-> (5,)
```

判断标准：

```text
forward 输入什么 shape，backward 返回的对应梯度就必须是什么 shape。

grad_x.shape == x.shape
grad_weight.shape == weight.shape
grad_bias.shape == bias.shape
```

## 易错点

1. `ctx.save_for_backward` 要保存 backward 真正要用的张量。

   本题应保存：

   ```python
   ctx.save_for_backward(x, weight, mask)
   ```

   不能误存成：

   ```python
   ctx.save_for_backward(x, weight, bias)
   ```

   否则 backward 里取出的第三个变量会变成 `bias`，导致 `grad_z = grad_output * mask` 实际变成乘 bias，数值错误。

2. `grad_output` 不是输出值本身。

   ```text
   grad_output = dL/dy
   grad_z      = dL/dz
   ```

3. ReLU 用 `*`，Linear 用 `@`。

   ```text
   ReLU 是逐元素函数，所以反向用逐元素乘法 *
   Linear 是矩阵乘法，所以反向用矩阵乘法 @
   ```

4. `grad_bias` 不是 `mask.sum(dim=0)`。

   ```python
   grad_bias = grad_z.sum(dim=0)
   ```

   因为 bias 的梯度要看真正传到 z 的梯度大小，而不仅仅是 ReLU 哪些位置通过。
