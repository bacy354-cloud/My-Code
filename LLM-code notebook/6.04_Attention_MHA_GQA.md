## 本节目标

理解并实现带 KV Cache 的 Multi-Head Attention / Grouped-Query Attention。

本节重点：

1. Attention 的作用级别是 token-token，不是 hidden 维度自己看自己。
2. `Q @ K.T` 用 `head_dim` 做匹配，得到 `[S_q, S_k]` 的注意力分数表。
3. `softmax(scores)` 决定看每个 token 的比例，`probs @ V` 才真正取内容。
4. MHA 切的是 `hidden_dim`，不是切 `seq_len`。
5. GQA 用更少的 K/V head，节省 KV Cache。
6. KV Cache 沿 token 维拼接，`repeat_kv` 沿 head 维扩展。

## 核心直觉

Attention 可以理解成：

```text
Q = 当前 token 发出的查询：我想找什么？
K = 每个 token 提供的标签：我有什么可被匹配？
V = 每个 token 真正提供的内容：我能贡献什么信息？
```

所以：

```text
Q @ K.T      -> 算“该看谁”
softmax      -> 把分数变成比例
probs @ V    -> 按比例取内容
```

最重要的一句话：

```text
计算材料来自 head_dim 维度；
作用结果落在 token-token 关系表上。
```

对单个 head：

```text
Q: [S_q, D]
K: [S_k, D]
K.T: [D, S_k]

Q @ K.T -> [S_q, S_k]
```

这里 `D` 被消掉，留下的是：

```text
第 i 行：第 i 个 query token 在看谁
第 j 列：它看第 j 个 key token 的分数
```

## MHA / MQA / GQA

假设：

```text
H = Query head 数
H_kv = Key/Value head 数
D = head_dim
```

三种结构：

| 结构 | Q head | K/V head | 直觉 |
|---|---:|---:|---|
| MHA | H | H | 每个 Q head 有自己的 K/V，表达强但 cache 大 |
| MQA | H | 1 | 所有 Q head 共享一个 K/V，cache 最小但表达弱 |
| GQA | H | H_kv | 一组 Q head 共享一个 K/V，工程折中 |

GQA 要求：

```text
H 能被 H_kv 整除
num_queries_per_kv = H // H_kv
```

例如：

```text
H = 8
H_kv = 2
num_queries_per_kv = 4
```

共享关系：

```text
Q0 Q1 Q2 Q3 -> KV0
Q4 Q5 Q6 Q7 -> KV1
```

`repeat_kv` 的效果：

```text
[KV0, KV1]
-> [KV0, KV0, KV0, KV0, KV1, KV1, KV1, KV1]
```

## Shape 主链路

以 notebook 测试参数为例：

```text
B = 2
S = 16
hidden_dim = 128
H = 4
H_kv = 2
D = hidden_dim // H = 32
```

输入：

```text
x: [B, S, hidden_dim]
x: [2, 16, 128]
```

线性投影：

```text
xq: [B, S, H * D]     = [2, 16, 128]
xk: [B, S, H_kv * D]  = [2, 16, 64]
xv: [B, S, H_kv * D]  = [2, 16, 64]
```

切多头：

```text
xq: [B, S, H * D]
 -> [B, S, H, D]
 -> [B, H, S, D]

xk: [B, S, H_kv * D]
 -> [B, S, H_kv, D]
 -> [B, H_kv, S, D]

xv: [B, S, H_kv * D]
 -> [B, S, H_kv, D]
 -> [B, H_kv, S, D]
```

代码：

```python
xq = xq.reshape(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)
xk = xk.reshape(batch_size, seq_len, self.num_kv_heads, self.head_dim).transpose(1, 2)
xv = xv.reshape(batch_size, seq_len, self.num_kv_heads, self.head_dim).transpose(1, 2)
```

注意：

```text
多头注意力切的是 hidden_dim，不是 seq_len。
S 个 token 不会被切开，每个 head 仍然能看完整 S 个 token。
```

## KV Cache

KV Cache 服务于 decoder-only LLM 的推理过程。

可以先这样记：

```text
prefill = 读题阶段，一次性处理 prompt，建立初始 KV Cache
decode = 写答案阶段，每次生成 1 个新 token，并追加 KV Cache
```

假设历史 cache 有 5 个 token：

```text
k_cache: [B, H_kv, 5, D]
v_cache: [B, H_kv, 5, D]
```

当前新 token 算出：

```text
xk: [B, H_kv, 1, D]
xv: [B, H_kv, 1, D]
```

拼接：

```python
xk = torch.cat([k_cache, xk], dim=2)
xv = torch.cat([v_cache, xv], dim=2)
```

shape：

```text
[B, H_kv, 5, D] + [B, H_kv, 1, D]
-> [B, H_kv, 6, D]
```

这里 `dim=2` 是 token 数维。

一句话：

```text
拼接是沿 token 维补历史；
repeat 是沿 head 维补头数。
```

所以顺序必须是：

```text
先 cache 拼接，再 repeat_kv。
```

否则如果先 repeat，再存 cache，cache 会变成：

```text
[B, H, S, D]
```

GQA 就失去节省显存的意义。

## Attention 计算

先把 GQA 的 K/V 临时扩到和 Q 一样的 head 数：

```python
xk = repeat_kv(xk, self.num_queries_per_kv)
xv = repeat_kv(xv, self.num_queries_per_kv)
```

shape：

```text
xq: [B, H, S_q, D]
xk: [B, H, S_k, D]
xv: [B, H, S_k, D]
```

算分数：

```python
scores = torch.matmul(xq, xk.transpose(2, 3)) / math.sqrt(self.head_dim)
```

shape：

```text
xq:             [B, H, S_q, D]
xk.T:           [B, H, D, S_k]
scores:         [B, H, S_q, S_k]
```

`scores` 还不是概率，只是 raw scores / logits。

再做：

```python
probs = torch.softmax(scores, dim=-1)
output = torch.matmul(probs, xv)
```

shape：

```text
probs:  [B, H, S_q, S_k]
xv:     [B, H, S_k, D]
output: [B, H, S_q, D]
```

这里 `S_k` 被消掉，因为是在对所有被看的 token 做加权求和。

## 合并多头

`output` 此时是：

```text
[B, H, S, D]
```

目标是：

```text
[B, S, H * D]
```

所以必须先换回 token 维和 head 维：

```python
output = output.transpose(1, 2).reshape(batch_size, seq_len, -1)
```

shape：

```text
[B, H, S, D]
-> [B, S, H, D]
-> [B, S, H * D]
```

不能直接：

```python
output = output.reshape(batch_size, seq_len, -1)
```

因为 `reshape` 只能拆分/合并维度，不能表达“把 H 和 S 换位置”。

## 最终代码

```python
import torch
import torch.nn as nn
import math

def repeat_kv(hidden_states: torch.Tensor, n_rep: int) -> torch.Tensor:
    batch, num_kv_heads, slen, head_dim = hidden_states.shape
    if n_rep == 1:
        return hidden_states
    hidden_states = hidden_states[:, :, None, :, :].expand(
        batch, num_kv_heads, n_rep, slen, head_dim
    )
    return hidden_states.reshape(batch, num_kv_heads * n_rep, slen, head_dim)

class GroupedQueryAttention(nn.Module):
    def __init__(self, hidden_dim: int, num_heads: int, num_kv_heads: int = None):
        super().__init__()
        self.hidden_dim = hidden_dim
        self.num_heads = num_heads
        self.num_kv_heads = num_kv_heads if num_kv_heads is not None else num_heads

        self.num_queries_per_kv = self.num_heads // self.num_kv_heads
        self.head_dim = hidden_dim // num_heads

        self.q_proj = nn.Linear(hidden_dim, num_heads * self.head_dim, bias=False)
        self.k_proj = nn.Linear(hidden_dim, self.num_kv_heads * self.head_dim, bias=False)
        self.v_proj = nn.Linear(hidden_dim, self.num_kv_heads * self.head_dim, bias=False)
        self.o_proj = nn.Linear(num_heads * self.head_dim, hidden_dim, bias=False)

    def forward(
        self,
        x: torch.Tensor,
        attention_mask: torch.Tensor = None,
        kv_cache: tuple[torch.Tensor, torch.Tensor] = None,
    ):
        batch_size, seq_len, _ = x.shape

        xq, xk, xv = self.q_proj(x), self.k_proj(x), self.v_proj(x)

        xq = xq.reshape(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)
        xk = xk.reshape(batch_size, seq_len, self.num_kv_heads, self.head_dim).transpose(1, 2)
        xv = xv.reshape(batch_size, seq_len, self.num_kv_heads, self.head_dim).transpose(1, 2)

        if kv_cache is not None:
            k_cache, v_cache = kv_cache
            xk = torch.cat([k_cache, xk], dim=2)
            xv = torch.cat([v_cache, xv], dim=2)

        new_kv_cache = (xk, xv)

        xk = repeat_kv(xk, self.num_queries_per_kv)
        xv = repeat_kv(xv, self.num_queries_per_kv)

        scores = torch.matmul(xq, xk.transpose(2, 3)) / math.sqrt(self.head_dim)

        if attention_mask is not None:
            scores = scores + attention_mask

        probs = torch.softmax(scores, dim=-1)
        output = torch.matmul(probs, xv)

        output = output.transpose(1, 2).reshape(batch_size, seq_len, -1)

        return self.o_proj(output), new_kv_cache
```

## 易错点

1. MHA/GQA 切的是 `hidden_dim`，不是切 `S`。
2. `Q @ K.T` 消掉的是 `D`，得到的是 token-token 分数表。
3. `scores` 不是概率，`softmax(scores, dim=-1)` 后才是概率权重。
4. `KV Cache` 沿 `dim=2` 拼接，因为第 2 维是 token 数。
5. `repeat_kv` 沿 head 维扩展，不沿 token 维扩展。
6. `repeat_kv` 要把 `[KV0, KV1]` 变成 `[KV0, KV0, KV1, KV1]`，不是 `[KV0, KV1, KV0, KV1]`。
7. 先 cache 拼接，再 `repeat_kv`，这样 cache 里存的是小的 `[B, H_kv, S, D]`。
8. `reshape` 不能替代 `transpose`，最后合并多头前必须先 `transpose(1, 2)`。
9. 解码阶段如果历史有 5 个 token，当前 token 也拼进去，则 `K/V` 的 `S_k = 6`。
