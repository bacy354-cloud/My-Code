## 本节目标

实现 LLaMA、Gemma 等大模型常用的 RMSNorm。

本节重点不是反向传播，而是理解：

1. RMSNorm 为什么比 LayerNorm 更省计算。
2. `weight` 作用在哪个维度。
3. `mean(dim=-1, keepdim=True)` 的 shape 变化。
4. 为什么 `_norm(x)` 里要先转 `float32`。
5. `nn.Parameter` 和 `.to()` 的作用。

## 核心直觉

LayerNorm 会做：

```text
减均值 -> 除以标准差 -> 乘 weight
```

RMSNorm 少了减均值：

```text
除以均方根 RMS -> 乘 weight
```

所以 RMSNorm 的核心是：

```text
不调整中心，只调整尺度。
```

它适合大模型，因为 Norm 会在每层反复调用，少算一步就能节省大量计算。

## RMS 是什么

RMS 是 Root Mean Square，均方根。

对一个 hidden 向量：

```text
x = [x1, x2, ..., xd]
```

RMS 为：

```text
RMS(x) = sqrt(mean(x^2) + eps)
```

其中 `eps` 是很小的数，用来防止除以 0。

RMSNorm 输出：

```text
y = x / RMS(x) * weight
```

其中：

```text
weight.shape = (hidden_size,)
```

## Shape 链条

大模型里输入通常是：

```text
x: (batch, seq_len, hidden_size)
```

例如：

```text
x: (4, 10, 768)
```

RMSNorm 对最后一维 hidden 做均方：

```python
variance = x.pow(2).mean(dim=-1, keepdim=True)
```

shape：

```text
x:        (4, 10, 768)
variance: (4, 10, 1)
```

`keepdim=True` 的作用：

```text
把 hidden 维度压成 1，而不是直接删掉。
```

这样后面可以广播：

```text
x(4,10,768) * rsqrt(variance(4,10,1))
-> (4,10,768)
```

`weight` 的 shape：

```text
weight: (768,)
```

会广播到：

```text
(4,10,768)
```

核心理解：

```text
每个 token 自己算一个 RMS。
weight 作用在每个 token 的 hidden 维度上，不作用在 token 位置上。
```

## 混合精度问题

输入可能是 `float16`，但 RMSNorm 里有平方：

```python
x.pow(2)
```

FP16 最大安全数值约为：

```text
65504
```

如果某个值大于 256：

```text
256^2 = 65536
```

就可能溢出成 `inf`，后面导致 `NaN`。

所以 `_norm(x)` 里先转：

```python
x_fp32 = x.float()
```

含义：

```text
输入可以是 FP16，但平方和 mean 这一步用 FP32 更安全。
```

最后再转回输入 dtype：

```python
return output.to(x.dtype)
```

## 可训练参数

普通 Tensor 不会自动成为模型参数：

```python
self.weight = torch.ones(hidden_size)
```

要让优化器更新它，需要写：

```python
self.weight = nn.Parameter(torch.ones(hidden_size))
```

这样它会进入：

```python
model.parameters()
```

一句话：

```text
nn.Parameter 是“把 Tensor 注册成可训练参数”。
```

## `.to()` 的作用

`.to()` 不只用来转类型，也能转设备。

常见用法：

```python
x.to(torch.float32)
x.to(torch.float16)
x.to("cuda")
x.to("cpu")
x.to(device="cuda", dtype=torch.float16)
```

在本节里：

```python
weight = self.weight.to(x.dtype)
```

意思是：

```text
让 weight 临时按输入 x 的 dtype 参与当前 forward。
```

但防溢出的关键不是这句，而是：

```python
x.float()
```

最关键的输出约束是：

```python
return output.to(x.dtype)
```

## 最终代码

```python
import torch
import torch.nn as nn

class RMSNorm(nn.Module):
    def __init__(self, hidden_size: int, eps: float = 1e-6):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(hidden_size))

    def _norm(self, x: torch.Tensor) -> torch.Tensor:
        x_fp32 = x.float()
        variance = x_fp32.pow(2).mean(dim=-1, keepdim=True)
        return x_fp32 * torch.rsqrt(variance + self.eps)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        output = self._norm(x) * self.weight
        return output.to(x.dtype)
```

也可以写成参考答案风格：

```python
def forward(self, x: torch.Tensor) -> torch.Tensor:
    weight = self.weight.to(x.dtype)
    return (weight * self._norm(x)).to(x.dtype)
```

## 易错点

1. 忘记 `keepdim=True`，导致广播不方便。
2. 误以为 `weight` 是作用在 token 维度，其实它作用在 hidden 维度。
3. 直接用 FP16 算平方，可能溢出。
4. 忘记把普通 Tensor 包成 `nn.Parameter`。
5. 忘记最后 `.to(x.dtype)`，导致输出 dtype 和输入不一致。
