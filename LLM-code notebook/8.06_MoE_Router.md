## 本节目标

理解 MoE 中最核心的 Router：每个 token 不再经过同一个完整 MLP，而是由 Router 选择少数几个 Expert。

本节重点：

1. MoE 主要替换 Transformer Block 里的 MLP 部分。
2. Router 负责给每个 token 选择专家。
3. Expert 可以先理解成多个独立的小 MLP。
4. Top-K Routing 表示每个 token 只激活概率最高的 K 个专家。
5. 正确顺序是先对所有专家做 Softmax，再取 Top-K。
6. Top-K 概率取出后要重新归一化，让每个 token 的专家权重和为 1。

## 核心直觉

上一节 LLaMA3 Block 的结构是：

```text
Norm -> Attention -> Add
Norm -> MLP -> Add
```

MoE 主要动的是第二个 MLP：

```text
Norm -> Attention -> Add
Norm -> MoE-MLP -> Add
```

Dense MLP 的做法：

```text
每个 token 都经过同一个完整 MLP。
```

MoE 的做法：

```text
准备多个 Expert。
每个 token 只选择少数几个 Expert。
```

所以 MoE 的优势是：

```text
总参数量可以很大，知识容量更大；
但每个 token 只激活一部分参数，实际计算量没有按总参数量同比例增加。
```

## MoE 三个组件

### Router

Router 是一个轻量门控网络。

它解决的问题是：

```text
当前 token 应该交给哪些 Expert 处理？
每个 Expert 的权重是多少？
```

### Experts

Expert 是多个独立的子网络。

在真实 LLM 里，Expert 通常是 MLP，可能是 SwiGLU MLP：

```text
Expert_i(x) = MLP_i(x)
```

### Top-K Routing

Top-K Routing 表示：

```text
每个 token 只选择概率最高的 K 个专家。
```

例如：

```text
num_experts = 8
top_k = 2
```

每个 token 只会激活 8 个专家中的 2 个。

## 核心数学机制

假设一个 token 的 hidden state 是：

```text
x shape = [hidden_size]
```

Router 先把它映射成所有专家的打分：

```text
h = x W_gate
h shape = [num_experts]
```

这里的 `h` 还不是概率，只是 logits。

接着对所有专家做 Softmax：

```text
p = softmax(h)
p shape = [num_experts]
```

然后取概率最高的 K 个专家：

```text
p_topk, idx_topk = TopK(p, K)
```

因为只保留了部分概率，Top-K 概率之和通常小于 1，所以需要重新归一化：

```text
w_i = p_i / sum(p_topk)
```

最后把被选中专家的输出加权融合：

```text
y = sum(w_i * Expert_i(x))
```

## Softmax Trap

本节最重要的易错点：

```text
正确：先对所有专家做 Softmax，再取 Top-K。
错误：先取 Top-K logits，再对局部 logits 做 Softmax。
```

原因：

```text
Router 要先让所有专家一起竞争，得到全局概率分布。
如果先截断 logits，就会丢掉没有被选中专家的全局比较信息。
```

正确链路：

```text
router_logits: [num_tokens, num_experts]
-> softmax over num_experts
-> routing_probs: [num_tokens, num_experts]
-> topk
-> routing_weights: [num_tokens, top_k]
-> selected_experts: [num_tokens, top_k]
-> re-normalize routing_weights
```

## Shape 主链路

假设 notebook 测试参数是：

```text
batch_size = 2
seq_len = 4
hidden_size = 16
num_experts = 8
top_k = 2
```

输入：

```text
hidden_states: [2, 4, 16]
```

展平 token：

```text
hidden_states.view(-1, hidden_size)
-> [8, 16]
```

这里：

```text
8 = batch_size * seq_len
```

Router 线性层：

```python
self.gate = nn.Linear(hidden_size, num_experts, bias=False)
```

所以：

```text
router_logits = self.gate(hidden_states)

[8, 16] -> [8, 8]
```

含义：

```text
8 个 token，每个 token 对 8 个专家各有一个打分。
```

全局 Softmax：

```text
routing_probs: [8, 8]
```

每一行加起来等于 1。

Top-K：

```text
routing_weights:  [8, 2]
selected_experts: [8, 2]
```

含义：

```text
routing_weights 存选中专家的权重。
selected_experts 存选中专家的编号。
```

重归一化后：

```text
routing_weights 每一行加起来等于 1。
```

最终 MoE 输出要恢复成：

```text
[8, 16] -> [2, 4, 16]
```

这样才能接回残差连接：

```text
out = h + MoE(RMSNorm(h))
```

## TODO 实现

### TODO 1：全量 Softmax

```python
routing_probs = F.softmax(router_logits.float(), dim=-1)
```

解释：

```text
router_logits.float()：用 FP32 计算 Softmax，数值更稳定。
dim=-1：最后一维是专家维度，所以在专家之间做 Softmax。
```

shape：

```text
[num_tokens, num_experts] -> [num_tokens, num_experts]
```

### TODO 2：Top-K 选专家

```python
routing_weights, selected_experts = torch.topk(
    routing_probs,
    self.top_k,
    dim=-1
)
```

解释：

```text
routing_weights：Top-K 概率值。
selected_experts：Top-K 专家编号。
```

shape：

```text
routing_weights:  [num_tokens, top_k]
selected_experts: [num_tokens, top_k]
```

### TODO 3：重归一化

```python
routing_weights = routing_weights / routing_weights.sum(dim=-1, keepdim=True)
```

解释：

```text
Top-K 概率截取后，每一行加起来通常不是 1。
重归一化后，每个 token 选中的 K 个专家权重之和重新变成 1。
```

`keepdim=True` 的作用：

```text
routing_weights: [8, 2]
routing_weights.sum(dim=-1, keepdim=True): [8, 1]

[8, 2] / [8, 1] 可以按 token 广播。
```

## 最终代码

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class TopKRouter(nn.Module):
    def __init__(self, hidden_size: int, num_experts: int, top_k: int):
        super().__init__()
        self.num_experts = num_experts
        self.top_k = top_k
        self.gate = nn.Linear(hidden_size, num_experts, bias=False)

    def forward(self, hidden_states: torch.Tensor):
        batch_size, seq_len, hidden_size = hidden_states.shape
        hidden_states = hidden_states.view(-1, hidden_size)

        router_logits = self.gate(hidden_states)

        routing_probs = F.softmax(router_logits.float(), dim=-1)

        routing_weights, selected_experts = torch.topk(
            routing_probs,
            self.top_k,
            dim=-1
        )

        routing_weights = routing_weights / routing_weights.sum(dim=-1, keepdim=True)
        routing_weights = routing_weights.to(hidden_states.dtype)

        return routing_weights, selected_experts
```

注意：

```text
notebook 里的 zeros 是占位代码。
完成 TODO 后要删掉，否则会把前面算好的 routing_weights 和 selected_experts 覆盖成全 0。
```

## SparseMoEBlock 聚合逻辑

Router 输出后，MoE Block 会遍历每个 Expert：

```python
for expert_idx, expert in enumerate(self.experts):
    token_idx, kth_expert = torch.where(selected_experts == expert_idx)
```

含义：

```text
找出哪些 token 选择了当前 expert。
```

然后取出这些 token：

```python
current_state = flat_hidden_states[token_idx]
```

送进当前 expert：

```python
current_output = expert(current_state)
```

取出对应权重：

```python
current_weight = routing_weights[token_idx, kth_expert].unsqueeze(-1)
```

加权累加回最终输出：

```python
final_hidden_states[token_idx] += current_output * current_weight
```

直觉：

```text
每个 Expert 只处理选择了自己的 token。
每个 token 最终得到 top_k 个 Expert 输出的加权和。
```

## scatter 和 gather 的位置

notebook 文字里提到了 `scatter` 和 `gather`，但当前 TODO 实现没有必须使用它们。

当前代码实际使用的是：

```text
torch.topk
torch.where
高级索引
unsqueeze
广播乘法
```

`gather` 常用于：

```text
已知专家编号 selected_experts，再从完整概率表 routing_probs 里取对应概率。
```

`scatter` 常用于：

```text
把 Top-K 的稀疏选择撒回完整专家维度。
```

所以你没看漏：

```text
这份 notebook 的描述范围比当前参考实现更宽。
通过本节测试只需要 topk + where + indexing。
```

## 易错点

1. 不要把占位的 `torch.zeros(...)` 留在 TODO 后面，否则会覆盖正确结果。
2. `selected_experts` 存的是专家编号，不是专家输出。
3. `routing_weights` 存的是专家权重，不是 logits。
4. Softmax 要在 `dim=-1` 上做，因为最后一维是专家维度。
5. Top-K 后必须重归一化，否则每个 token 的专家权重和不是 1。
6. `routing_weights.sum(dim=-1, keepdim=True)` 的 shape 是 `[num_tokens, 1]`。
7. `current_weight.unsqueeze(-1)` 是为了从 `[n]` 变成 `[n, 1]`，方便乘 `[n, hidden_size]`。
