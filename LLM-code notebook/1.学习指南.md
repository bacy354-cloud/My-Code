# LLM-Algo-LeetCode 学习指南

## 总体认知

LLM-Algo-LeetCode 不是普通的“大模型科普课”，它更像一套 LLM 算法工程刷题册。它把大模型底层常见考点拆成 Jupyter Notebook 题目：你读题、补 TODO、跑测试、对答案。它适合用来建立“我真的会写出来”的能力，而不是只停留在概念背诵。

这门课最重要的目标不是一次学完全部章节，而是形成三种能力：

- 能用 PyTorch 写出 Transformer 里的核心算子和训练组件。
- 能估算参数量、FLOPs、显存、通信开销，知道模型为什么跑不动。
- 能理解 CUDA/Triton、FlashAttention、PagedAttention、ZeRO、并行训练等系统概念，为后续科研和工程复现打基础。

你的本机环境是 CPU-only，所以策略是：本机完成 Chapter 0、Chapter 1、Chapter 2 的大部分内容；Chapter 3 和强 GPU 题先建立认知，真正实践放到云 GPU。

## 知识链路

这门课的知识链路可以按“从张量到系统”的顺序理解：

```text
PyTorch 基础
  -> Tensor / shape / broadcasting / autograd
  -> 基础层：Linear / Embedding / Norm / Activation
  -> Transformer 部件：RoPE / Attention / MLP / Block
  -> 训练部件：SFT / LoRA / RLHF / DPO / Scheduler
  -> 推理部件：KV Cache / PagedAttention / Speculative Decoding / 量化
  -> 系统部件：显存估算 / ZeRO / Tensor Parallel / Pipeline Parallel
  -> GPU 算子：Triton / CUDA / FlashAttention
```

学习时不要把每个 notebook 当孤立题。每做一道题，都问自己三个问题：

- 这个组件解决什么问题？
- 输入输出 shape 是什么？
- 它和 LLM 的训练、推理、显存或速度有什么关系？

## 阶段式学习路径

### 阶段一：刷题工作流与 PyTorch 基础

范围：`00_Prerequisites` + `02_PyTorch_Algorithms/00_PyTorch_Warmup.ipynb`

目标：先熟悉 notebook 题目结构、`TODO`、测试函数、答案区边界。不要一上来冲 MoE、RLHF、CUDA。

最小成果物：

- 能打开 Jupyter Lab。
- 能运行 `00_PyTorch_Warmup.ipynb`。
- 能手写 Tensor reshape、Embedding 查表、Linear 前向和反向。
- 能跑通官方测试脚本。

已完成笔记：

- [[2.00_PyTorch_Warmup|00_PyTorch_Warmup]]
- [[3.01_RMSNorm|01_RMSNorm]]
- [[4.02_SwiGLU|02_SwiGLU]]

验证命令：

```powershell
$env:PYTHONUTF8='1'
$env:PYTHONIOENCODING='utf-8'
python test_notebook_answers.py 02_PyTorch_Algorithms/00_PyTorch_Warmup.ipynb --mode answer
```

### 阶段二：硬件、显存和系统直觉

范围：`01_Hardware_Math_and_Systems`

目标：建立“模型为什么占显存、为什么慢、为什么要并行”的直觉。

重点顺序：

1. 数据类型与精度：FP32、FP16、BF16、INT8。
2. 参数量和 FLOPs：先会估算，不追求一次背熟公式。
3. GPU 架构和显存：知道显存主要花在参数、梯度、优化器状态、激活值、KV Cache。
4. 通信拓扑和 ZeRO：先理解切分对象，再理解工程实现。

最小成果物：

- 写一页笔记：`一个 7B 模型训练和推理时显存花在哪里`。
- 能解释为什么本机 CPU 适合学习结构，但不适合跑 CUDA/Triton。

### 阶段三：PyTorch 实现 LLM 核心模块

范围：`02_PyTorch_Algorithms/01` 到 `02_PyTorch_Algorithms/16` 优先。

推荐顺序：

1. RMSNorm、SwiGLU、RoPE。
2. MHA/GQA、LLaMA Block。
3. MoE Router、MoE Load Balancing。
4. LoRA、SFT Training Loop。
5. RLHF/PPO、DPO。
6. Decoding Strategies。

最小成果物：

- 每个模块至少完成一个 notebook 的 TODO。
- 每个 notebook 结束后写 5 行记录：输入 shape、输出 shape、核心公式、容易错的点、和 LLM 的关系。

本阶段不要做：

- 不要同时追求所有题全会。
- 不要在没懂 shape 的情况下硬背公式。
- 不要急着跑大模型权重。

### 阶段四：推理优化、量化和并行训练认知

范围：`02_PyTorch_Algorithms/17` 到 `25`

目标：把推理服务和训练系统里的关键词串起来。

重点：

- PagedAttention：为什么 KV Cache 需要分页管理。
- Speculative Decoding：为什么小模型草稿能加速大模型生成。
- 量化：W8A16、QLoRA、4bit 量化分别在省什么。
- Gradient Checkpointing：用计算换显存。
- ZeRO、Tensor Parallel、Pipeline Parallel：切参数、切张量、切层的区别。

最小成果物：

- 做一张表：`优化方法 -> 解决的问题 -> 牺牲什么 -> 适用场景`。

### 阶段五：CUDA/Triton 云端实践

范围：`03_CUDA_and_Triton_Kernels`

本机边界：本机无 NVIDIA CUDA，不建议硬跑。先阅读题目和图解，理解算子为什么能快。

云 GPU 实践顺序：

1. 先用课程提供的 Colab / ModelScope 入口。
2. 只挑 1-2 个入门题：Triton 基础算子、FlashAttention 简化版。
3. 每次只记录一个问题：比如 shared memory、block size、memory coalescing、benchmark。

最小成果物：

- 一份云端运行截图或日志。
- 一页笔记：`FlashAttention 为什么省显存`。

## 使用规则

- 每次学习只打开一个 notebook，不要十几个标签页并行。
- 优先自己补 TODO，再看 STOP HERE 后面的答案。
- 跑不通时先检查 shape，再检查 dtype，再检查 device。
- 每阶段都要有最小成果物：笔记、测试通过记录、或一段可运行代码。
- 遇到 CUDA/Triton 内容，本机只读；真正运行放云 GPU。

## 和 Happy-LLM 的配合

- Happy-LLM 负责建立 LLM 全景认知和从零搭模型。
- LLM-Algo-LeetCode 负责把关键模块刷到“能写、能测、能解释”。
- 学到 Happy-LLM 第 2 章 Transformer 时，可以同步刷 LLM-Algo 的 RMSNorm、RoPE、Attention。
- 学到 Happy-LLM 第 5/6 章训练时，可以同步刷 LoRA、SFT、DPO、ZeRO。
