## 本节目标

理解并实现 LLaMA / Qwen / Mistral 等大模型常用的 SwiGLU MLP。

本节重点：

1. SwiGLU 的 gate / up 两路结构。
2. 为什么中间维度是 `8/3 * hidden_size`。
3. 为什么还要对齐到 `multiple_of=256`。
4. `gate_up_proj` 为什么要融合两个投影矩阵。
5. `torch.chunk`、`F.silu`、`nn.Linear` 分别属于哪类函数。

## 核心直觉

普通 MLP：

```text
x -> up_proj -> activation -> down_proj
```

SwiGLU 是“分-合”结构：

```text
x -> gate_proj -> SiLU -> *
x -> up_proj   ---------^
                   |
               down_proj
```

可以记成：

```text
分：分成 gate 路和 up 路
合：SiLU(gate) * up
总：down_proj 投回 hidden_size
```

其中：

```text
gate 路：生成动态门控向量
up 路：生成候选内容向量
```

`gate` 可以理解成“动态权重向量”，但它不是固定可训练参数。

真正可训练的是：

```text
gate_proj.weight
up_proj.weight
down_proj.weight
```

`gate = gate_proj(x)` 是当前输入动态算出来的控制信号。

## SiLU 和 Swish

Swish 一般写成：

```text
Swish(x) = x * sigmoid(beta * x)
```

当：

```text
beta = 1
```

就是：

```text
x * sigmoid(x)
```

这就是 PyTorch 里的 SiLU：

```python
F.silu(x)
```

所以本节可以认为：

```text
Swish(beta=1) = SiLU
```

在 SwiGLU 中：

```python
gate = gate_proj(x)
gate_signal = F.silu(gate)
```

注意这里 SiLU 作用的是：

```text
gate_proj(x) 的结果
```

不是原始输入 `x`。

## Shape 链条

设：

```text
x: (batch, seq, hidden_size)
```

SwiGLU 中间维度：

```text
intermediate_size
```

朴素版本：

```text
gate = gate_proj(x) -> (batch, seq, intermediate_size)
up   = up_proj(x)   -> (batch, seq, intermediate_size)

hidden = F.silu(gate) * up
       -> (batch, seq, intermediate_size)

out = down_proj(hidden)
    -> (batch, seq, hidden_size)
```

工业融合版本：

```python
gate_up = gate_up_proj(x)
gate, up = torch.chunk(gate_up, 2, dim=-1)
hidden = F.silu(gate) * up
out = down_proj(hidden)
```

例子：

```text
x: (2, 10, 4096)
intermediate_size: 11008
```

则：

```text
gate_up: (2, 10, 22016)
gate:    (2, 10, 11008)
up:      (2, 10, 11008)
hidden:  (2, 10, 11008)
out:     (2, 10, 4096)
```

## 为什么是 8/3

普通 Transformer MLP 通常是：

```text
d -> 4d -> d
```

参数量：

```text
up_proj:   d * 4d = 4d^2
down_proj: 4d * d = 4d^2
总共:      8d^2
```

SwiGLU 有三套矩阵：

```text
gate_proj: d -> h
up_proj:   d -> h
down_proj: h -> d
```

参数量：

```text
d*h + d*h + h*d = 3dh
```

为了和普通 MLP 参数量对齐：

```text
3dh = 8d^2
```

所以：

```text
h = 8/3 d
```

一句话：

```text
SwiGLU 多了一条 gate 投影，所以不能继续用 4d；为了参数量对齐，中间维度降到 8/3d。
```

## 为什么要对齐到 256

理论值：

```python
intermediate_size = int(hidden_size * 8 / 3)
```

例如：

```text
hidden_size = 4096
4096 * 8 / 3 = 10922.66
int(...) = 10922
```

但真实模型通常会把它向上对齐到 `multiple_of=256` 的倍数：

```text
256 * 42 = 10752
256 * 43 = 11008
```

所以最终：

```text
aligned_size = 11008
```

原因：

```text
矩阵维度更适合硬件计算；
张量并行时更容易平均切分；
Tensor Core / 内存访问更友好。
```

向上对齐公式：

```python
aligned_size = ((intermediate_size + multiple_of - 1) // multiple_of) * multiple_of
```

其中：

```text
// 是整数除法
int(...) 是去掉小数部分，不是四舍五入
```

## `torch.chunk`

`torch.chunk(input, chunks, dim=0)` 用来：

```text
沿指定维度，把 Tensor 尽量平均切成 chunks 份。
```

本节：

```python
gate, up = torch.chunk(gate_up, 2, dim=-1)
```

意思是：

```text
把 gate_up 沿最后一维切成两半。
```

如果：

```text
gate_up: (2, 10, 22016)
```

那么：

```text
gate: (2, 10, 11008)
up:   (2, 10, 11008)
```

`dim=-1` 表示最后一维。

## `torch` / `nn` / `F` 的分工

不用硬背，先按类型记：

| 类型 | 位置 | 例子 |
|---|---|---|
| 通用 Tensor 操作 | `torch.xxx` | `torch.chunk`, `torch.sqrt`, `torch.rsqrt` |
| 神经网络模块 / 层 | `nn.xxx` | `nn.Linear`, `nn.Embedding`, `nn.Module` |
| 神经网络函数 | `F.xxx` | `F.silu`, `F.relu`, `F.softmax`, `F.linear` |

规则：

```text
要注册成模型层，用 nn。
只是 forward 里临时算一下，用 F。
通用张量数学和切分拼接，用 torch。
```

本节：

```python
self.gate_up_proj = nn.Linear(...)
hidden = F.silu(gate) * up
gate, up = torch.chunk(...)
```

## 最终代码

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

def calculate_intermediate_size(hidden_size: int, multiple_of: int = 256):
    intermediate_size = int(hidden_size * 8 / 3)
    aligned_size = ((intermediate_size + multiple_of - 1) // multiple_of) * multiple_of
    return aligned_size

class SwiGLU_MLP(nn.Module):
    def __init__(self, hidden_size: int, intermediate_size: int):
        super().__init__()
        self.gate_up_proj = nn.Linear(
            hidden_size,
            2 * intermediate_size,
            bias=False,
        )
        self.down_proj = nn.Linear(
            intermediate_size,
            hidden_size,
            bias=False,
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        gate_up = self.gate_up_proj(x)
        gate, up = torch.chunk(gate_up, 2, dim=-1)
        hidden = F.silu(gate) * up
        return self.down_proj(hidden)
```

## 易错点

1. `nn.Linear` 不写 `bias=False` 时，默认 `bias=True`。
2. `torch.chunk(..., dim=-1)` 要沿最后一维切，不要切 batch 或 seq 维。
3. `F.silu(gate)` 里的 `gate` 是 `gate_proj(x)`，不是原始 x。
4. `gate` 是动态门控向量，不是固定的 `nn.Parameter`。
5. `int(...)` 是截断小数，不是四舍五入。
6. 参数量对齐的关键是 `3dh = 8d^2`。
